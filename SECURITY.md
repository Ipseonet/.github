# Security Policy

This policy applies to all repositories in the **IPSEONET** organization
(the `@ipseonet` npm scope and related projects), unless a repository provides
its own `SECURITY.md`.

## Reporting a Vulnerability

Please report security issues **privately** — do not open a public issue.

- **Preferred:** GitHub [private vulnerability reporting](https://docs.github.com/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
  — open the affected repository, go to the **Security** tab, and choose
  **Report a vulnerability**.
- **Email:** [security@ipseonet.com](mailto:security@ipseonet.com)

Please include a description of the issue, the affected package and version,
and steps to reproduce (a minimal proof of concept is ideal).

## What to expect

- **Acknowledgement** within **3 business days**.
- An assessment and, where confirmed, a **fix or mitigation within 30 days**,
  with disclosure timing coordinated with you.
- Credit for the report if you would like it.

## Supported Versions

The latest published minor release of each package receives security fixes.
Older versions may not be patched — please upgrade to the latest release.

## Scope

Vulnerabilities in IPSEONET-published code and its release/publish pipeline are
in scope. Issues in third-party dependencies should generally be reported
upstream; if a dependency issue affects one of our packages, let us know and we
will coordinate an update.
